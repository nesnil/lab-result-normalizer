# 参考区间表

阶段 5（异常判定）查本表。所有区间**基于标准单位**（与 `loinc_mapping.md` / `unit_conversion.md` 一致），即先换算到 std_unit 再比对。

## 判定规则

1. **区间来源优先级**：
   - 若报告自带 `raw_ref_range` 且单位与 std_unit 一致 → 优先采用报告区间（尊重各实验室方法学差异）。
   - 否则用本表标准区间。
   - 报告区间与本表标准区间**明显冲突**（如上限差异>50%）时，以本表为准，并加 warnings（参考区间冲突，已采用标准区间）。
2. **分层选择**：按性别、年龄选对应行；缺性别/年龄信息时用"通用成人"行，并加 warnings（缺分层信息，按通用成人区间）。
3. **判定 flag**：`std_value < 下限` → `L`(↓)；`> 上限` → `H`(↑)；区间内 → `N`。
4. 定性项不在本表（无区间），flag 置 `N/A`。
5. 危急值另查 `abnormal_flag_rules.md`，优先级高于普通 H/L。

> ⚠️ 参考区间因检测系统、地区、人群而异。本表为**通用成人参考值（标注为 待核验）**，演示与数据治理可用；实际部署应替换为目标实验室的方法学区间。儿童/孕妇区间未纳入 v1（遇到加 warnings）。

## 特殊判定说明

- **HDL-C**：医学意义上"越高越好"，临床异常为"偏低"（<1.0 mmol/L 标 L）；上限设大值仅为不误报 H。
- **TC / LDL-C / TG**：采用"合适水平上限"，超过即标 H（升高），下限 0。
- **BASO#/EO#** 下限接近 0，低值一般不报 L（除非为 0 且有临床意义），实现上下限为 0 即不判 L。
- 缺乏性别信息且该项按性别分层时：取男女区间的**并集**（下限取较低、上限取较高）做宽松判定，并加 warnings。
