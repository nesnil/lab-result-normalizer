# 资料抽取规则

阶段 1（资料解析与要素归集）查本文件。定义从各类住院资料中抽取出院小结要素的规则、来源标注约定，以及**从输入资料到出院小结各字段的映射与提炼逻辑**（依真实案例校准）。

> 输入 = 病案首页 + 入院记录 + 术前小结 + 手术记录 + 病程记录；输出 = 出院记录（出院小结）。本 skill 的核心即"据前五者，提炼出后者"。

## 输入资料类型与对应要素（含常见字段位置）

| 资料类型 | 可抽取要素 | 真实案例中的关键段落 |
|---|---|---|
| **病案首页** | 姓名/性别/年龄/婚姻/职业、住院号、科别、入出院日期、门急诊诊断、出院诊断+疾病编码、手术及操作表（术式/日期/术者/切口类别/愈合等级/麻醉） | 首页人口学区、出院诊断表、手术及操作编码表 |
| **入院记录** | 基本信息、入院日期、主诉、现病史、既往史、个人史、查体（生命体征+全身）、专科情况、入院辅检、入院诊断 | "主诉""现病史""既往史""体格检查""专科情况""实验室及器械检查""入院诊断" |
| **术前小结** | 病历摘要、术前诊断、诊断依据、手术指征、拟行手术、拟施术者、拟行麻醉、手术级别、术前实验室检查 | "病历摘要""术前诊断""拟行手术""手术指征" |
| **手术记录** | 手术日期、起止时间、术前/术后诊断、规范术式名、术者/助手、麻醉方法、体位、术中所见、手术过程（入路/内固定/植入物品牌）、出血量、引流 | "手术名称""术中所见""手术过程""术中出血量" |
| **病程记录** | 首程病例特点、诊断依据、鉴别诊断、诊疗计划、入院评估（VTE-Caprini 评分/疼痛评分/预防措施）、查房记录、术后病程、术后复查结果、出院前情况、出院医嘱、院外随访安排 | "首次病程记录""入院评估""查房记录""术后病程""出院日/末次查房记录" |
| 医嘱单（如提供） | 用药（名称/用法）、康复医嘱、VTE 预防医嘱 | — |
| 检验/检查结果（如单列） | 关键化验、影像复查、出院前重要结果、影像号 | — |

## 出院小结字段 ← 输入资料 的提炼映射（核心逻辑，依真实案例校准）

> 原则：**提炼 ≠ 搬运**。入院情况要压缩，诊疗经过要筛选关键+异常，医嘱要按术式补专科内容。

| 出院小结字段 | 来源 | 提炼逻辑 |
|---|---|---|
| 页眉/人口学 | 病案首页 / 入院记录页眉 | 直接取 |
| 入院诊断 | 入院记录"入院诊断" | 取，保"1. 2."编号，主诊断在前 |
| 手术名称+日期 | 手术记录 / 首页手术表 | 取规范术式名+日期（阶段2规范化）；非手术患者无此行 |
| 出院诊断 | 首页出院诊断 / 术后诊断 | 通常=入院诊断；病程有修正则以病程为准（触发 D6） |
| 入院情况·主诉 | 入院记录"主诉" | 句式：`患者因"…"入院。` |
| 入院情况·既往 | 入院记录"既往史" | **压缩**：仅留关键（如高血压+用药），略去成串"否认…史" |
| 入院情况·查体/专科 | 入院记录"体格检查""专科情况" | 留生命体征+患肢阳性体征；全身阴性项高度压缩 |
| 入院情况·辅检 | 入院记录"实验室及器械检查" | 留入院时关键影像 |
| 诊疗经过·入院处置 | 病程首程"诊疗计划" | 套本院固定句（见 discharge_template §三·2） |
| 诊疗经过·完善检查 | 病程/检验检查结果 | 按日期全程扫描，术前关键检查 + 异常结果必留；检查名、科室、日期、时间戳、结果原文逐字保留 |
| 诊疗经过·手术 | 手术记录 | 麻醉+术式+术中所见原文摘录 |
| 诊疗经过·术后恢复复查 | 病程（术后病程 + 出院日前复查记录） | 术后处置套句 + 每次复查关键指标；检验、影像、血管超声异常发现不得丢 |
| 诊疗经过·出院决策 | 病程（出院日/末次查房记录） | 取"病情平稳…予出院"等原文依据；未见原文才标缺失 |
| 出院情况 | 病程（出院日/末次查房记录的出院病情情况/查体） | 全身+专科+伤口三段；源文有则直接写入，不得标"未见原始记录" |
| 出院医嘱·VTE/负重/康复 | 原始出院记录 / 医嘱单 / 病程记录出院日或末次查房的"出院医嘱" | 只写原文已有医嘱；病程记录中明确"出院医嘱："即为合法首选来源，不得降级为"参考" |
| 出院医嘱·复诊 | 原始出院记录 / 医嘱单 / 病程记录出院日或末次查房 | 原文有医生、门诊时间、"不适随诊"即照写；无信息 → 【待补充：复诊医生及时间】 |
| 随访·日期/内容/影像号 | 病程记录出院日或末次查房 / 检查报告 | "院外随访复查日期/内容"若在病程记录出现即直接采用；影像号源自检查报告，无则【待补充】 |

## 长病程记录全程覆盖规则（防止只抽前半段）

病程记录通常很长，且出院医嘱、出院查体、术后复查结果常位于文档末段。阶段 1 必须先做"时间线扫描"，再抽字段：

1. **建立病程时间线**：按记录日期/时间列出所有病程条目，至少记录首条日期、手术日期、术后各条日期、最后一条日期。
2. **核对末段覆盖**：最后一条病程日期原则上应为出院日或出院前 1 日；若未扫描到末段，阶段 5 必须提示"长病程记录可能截断/漏抽"。
3. **出院日（末次）查房记录优先**：末次查房若含以下关键词，对应内容直接作为字段来源：
   - `出院医嘱：` → 出院医嘱正文；
   - `院外随访复查日期：` / `院外随访复查内容：` → 随访区块；
   - `出院病情情况` / 出院前查体段 → 出院情况；
   - `患者目前病情平稳` / `可予出院` / `予出院` → 出院决策。
   **禁止**出现"参考中逐字复述了末次查房原文，但正文仍写【待补充：原始医嘱未提供】"这种自相矛盾输出。
4. **术后复查逐条归集**：手术日期之后所有检验、影像、超声、血管检查结果均进入 `treatment_course.followup_labs[]` / `postop_imaging[]`，异常值与关键影像结论必留。例如凝血、D-二聚体、CRP、血沉、术后 X 线、下肢血管超声等。
5. **不要遇到"出院记录/医嘱未提供"就停止**：本 skill 的输入常常只有首页、入院记录、术前小结、手术记录、病程记录；病程记录中的出院日查房同样可提供正式出院医嘱与随访安排。

## 多来源冲突与 OCR 噪声处理

抽取阶段遇到同一事实多种写法时，不得随机采信单一版本。记录 `source_candidates[]`，并按以下规则处理：

1. **多数/合理版本优先**：同一项目若一个来源写"S/CD"、另一来源写"S/CO"，且医学标准单位为 S/CO，则正文优先写"S/CO"，质控注明"S/CD 疑似 OCR 噪声/来源冲突，已按可核实来源采用 S/CO"。
2. **数值末位疑似箭头误并入**：如同一项目在不同来源表现为 `3.62↑mg/L` 与 `3.621mg/L`，或临近文本有异常箭头而数值末尾多出孤立数字，正文采用可核实原文/多数版本；质控标 `ocr_suspect=true`，提示医生核对。
3. **不确定则不清洗成事实**：若只有一个疑似噪声版本、无法判断，应正文保留原文并标【待核对】，质控列出"疑似 OCR 噪声"。
4. **冲突不可静默消失**：单位、数值、侧别、日期、药名、评分冲突均需进阶段 5 质控；即使正文选用了较合理版本，也要说明依据。

## 输入资料类型与对应要素（原始）

| 资料类型 | 可抽取要素 |
|---|---|
| 护理/评分记录 | 疼痛评分、功能评分、Barthel 指数、VTE-Caprini 评分等（如有；真实病程记录"入院评估"中即含 Caprini 评分与预防措施） |

## 抽取约定

1. **保留来源**：每个抽取要素记录 `source`（如"手术记录"），多来源冲突时全部保留并在阶段 5 提示。
2. **保留原文**：关键术语保留原始表述 `raw_*`，规范化在阶段 2 进行。
3. **缺失即 null**：抽不到的要素记 `null`，不推测、不补全，留待阶段 5 质控。
3b. **🔴 关键值逐字留证 `source_verbatim`**：**药物名称、Caprini/各类评分、数值、日期、次数、部位**等敏感值，抽取时**连同原文原句一并保留**（`source_verbatim`），写入正文时逐字比对，**严禁改写、近似或用同类替换**（"那屈肝素钙"≠"依诺肝素"，评分 8≠7）。这些值只能"复制"，不能"生成"。
3c. **🔴 事件必须有原文出处**：MDT/会诊/术前讨论/并发症/处置等**事件**，只有原文明确记载才可写入；原文没有的事件（哪怕"骨科惯例"）一律不写，不编造。
3d. **🔴 检验检查逐条逐字，禁摘要**：抽取检验检查时**逐条保留**（检查名+科室+日期+时间戳+结果原文），**禁止归并为"未见明显异常"**；异常值（血沉↑、心超"左房增大伴二尖瓣反流"、脂肪肝、D-二聚体↑、CRP、WBC↑等）**必须逐条留存**，绝不静默丢弃。术前、术后复查结果同此。
3e. **🔴 专科查体整段抽取**：专科情况须**整段**抽取（含脊柱/骨盆等阴性描述句 + 患肢阳性体征），不得只截取患肢部分而漏掉前置阴性句。
3f. **🔴 侧别矛盾不静默纠正**：检查名称侧别与结论/诊断侧别不一致时，**原文原样保留**（记 `raw_*` + `laterality_conflict=true`），进阶段 5 质控，绝不改写侧别去"消除"矛盾。
3g. **数值前禁模糊词**：源文为确切值（如"20ml"）时逐字照写，不得加"约/大约/左右"。
3h. **🔴 查体与术后处置不得同义改写/省略**：短句可直接复制，不要改写成泛化表述；如原文为"颈椎、胸椎、腰椎生理弯曲存在"，不得写成"脊柱无明显畸形"；原文有"生理反射存在，病理反射未引出""予吸氧"等，均应逐句保留。
3i. **推断性规范化不入正文事实**：若手术记录未直接写某入路名称，仅描述切口和解剖间隙，正文保留描述原文；规范化入路只写入 `std_approach` 或质控说明，不能替代原文成为正文事实。
4. **日期归一**：各种日期写法（YYYY-MM-DD、YYYY年M月D日等）归一为 ISO 日期；无法确定的标注。
5. **侧别识别**：明确记录患肢侧别（左/右/双侧）；不同资料侧别矛盾时进质控（D5）。
6. **去标识**：演示/脱敏场景下保留字段结构，姓名等可用占位（如"患者"）。

## 与逻辑校验的衔接

阶段 5 的逻辑校验（D1–D13，定义见 `discharge_template.md`）所需字段在此阶段确保抽取：入院/出院/手术日期、住院天数、出院诊断与术式的部位与侧别、术中所见部位、诊疗经过引用的检查日期、病程末段覆盖状态、多来源冲突与 OCR 噪声候选。

> 待医生补充：本院各类资料的常见格式/字段名，便于提升抽取命中率（如本院手术记录的固定段落结构）。
