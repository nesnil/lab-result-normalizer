# 输出格式规范

阶段 5 输出查本文件。输出顺序：**① 人读出院小结草稿（核心）→ ② 质控清单 → ③（可选）结构化要素 JSON**。

## 一、人读出院小结草稿（核心产物）

按 `discharge_template.md` 的章节顺序与**真实本院版式**成文（诊断三联并排、出院转归/伤口愈合固定字段、随访区块），规范临床语言，可直接供医生审核修改。要点：
- 缺失字段写占位 **【待补充：xxx】**，不编造。
- 诊疗经过为**叙事式一大段**（入院处置→完善检查→手术→术后恢复复查→出院决策），用本院固定套句（见 `discharge_template.md` §三·2）。
- 出院医嘱、患肢功能、负重、VTE、随访安排只写原始材料中已有内容；`rehab_vte_protocols.md` 的分阶段康复/VTE/随访方案默认只进入质控清单"可选建议"，不并入正文。
- 文末固定附免责声明。

### ⚠ 排版规范（务必遵守——用户会把文本贴入 Word，排版须贴合真实病历，逐条照做）

1. **每个字段独立成行，一行只放一条信息**。禁止把多个字段挤在一行（错误："姓名：X 性别：男 …住院号：Y 姓名：X 年龄：70"；正确：拆成两行，页眉信息一行、人口学信息一行）。
2. **诊断三联左右对齐**：诊断在左、日期在右，中间用**全角空格填充对齐**，三行的"日期"起始列尽量对齐。格式：
   ```
   入院诊断：1.踝关节骨折；2.高血压病1级(高危)　　　　　　入院日期：2026-06-14
   手术名称：1.右踝关节骨折切开复位内固定术　　　　　　　手术日期：2026-06-18
   出院诊断：1.踝关节骨折；2.高血压病1级(高危)　　　　　　出院日期：2026-06-25
   ```
3. **章节标题独占一行**（`入院情况：` `诊疗经过：` `出院情况：` `出院医嘱：` 各自成行，内容另起）。
4. **出院情况的固定字段行**：`出院转归：好转　　　　伤口愈合：Ⅰ/未拆线` 用全角空格对齐；`出院病情情况：` 另起一行。
5. **出院医嘱多条时用中文数字编号**（`1、` `2、` `3、`…），**不要用 `•`/`-`/`*` 等符号**。若原文医嘱本身是一整段，可保留一段式原文，不强行拆号。骨科分阶段负重只有在用户明确要求补充建议时才另列，不能混入原文医嘱。
6. **随访区块每字段独立成行**：
   ```
   院外随访复查日期：2026-07-23
   院外随访复查内容：复查
   是否推荐至其他科室诊治：【待补充】（仅源文明确"否/是"时填写）
   X光片号：__　CT号：__　MRI号：__　病理检验号：__
   签收人：___
   ```
7. 全篇用**全角标点**，段落之间空一行；不使用 Markdown 的 `#` 标题、`|` 表格、`**加粗**` 等标记（贴进 Word 会残留符号）——直接用纯文本 + 中文冒号 + 全角空格排版。

> 目的：让弱模型（DeepSeek 等）输出的纯文本，用户直接全选贴进 Word 即接近正式病历版式，无需二次手工排版。

格式示例（**依真实踝关节骨折案例校准**）——**AI 应严格照此**：①排版（字段分行、诊断三联对齐、中文编号、无 Markdown）；②**忠实度（医嘱只写原文有的，不塞分阶段方案；评分/药名逐字取自原文；无信息处留【待补充】）**。

⚠️ 注意本示例的**忠实性要点**（对比曾出现的错误）：
- 诊疗经过里的 **Caprini 评分、抗凝药名**均**逐字取自病程记录原文**（评分是几分写几分，药名原文是什么写什么），示例中用 `【取自病程原文：X】` 标示"必须来自原文"之处——实际输出直接填原文值。
- **出院医嘱只保留原文医嘱**，**不**写术后 0–2 周/2–6 周分阶段方案（那是知识库通用建议，进 💡 质控清单，不进正文）。
- 原文没有 MDT/健康宣教/额外药物 → **一概不写**。

> 上例要点：
> - 出院医嘱**仅一段、照搬原文风格**，**未**出现分阶段负重方案——因为原文没有。分阶段康复/VTE 只进质控 💡「可选建议（非原文）」，或用户明确要求时另列 `【AI 补充建议·非原始病历内容】` 段。
> - **诊疗经过的检验检查须逐条逐字**，原文若有异常（血沉↑、心超异常、脂肪肝、D-二聚体↑、CRP、WBC↑等）**一条都不能概括成"未见明显异常"丢掉**。
> - **辅检侧别照原文抄**，若"检查名侧别 vs 结论侧别"矛盾（如"右侧膝片：左侧退变"），**正文保留原文、进质控清单**，绝不静默改成一致。
> - **出院日/末次查房记录**中的出院医嘱、出院查体、院外随访是合法首选来源，源文有则直接写入，不得标待补充。
> - **OCR 噪声**如 `3.621mg/L` 疑似 `3.62↑mg/L`、`S/CD` vs `S/CO`，正文按可核实来源写，质控提示医生核对。
> - **文末 `门诊病历已交至患者或家属。`、`签收人：___` 与 `医师签名：___` 固定保留**，无信息则占位。
> - 数值前不加"约"（源文"20ml"就写"20ml"，不写"约20ml"）。

## 二、质控清单（必出）

分级列出，便于医生快速核对：

| 级别 | 类型 | 示例 |
|---|---|---|
| 🔴 必填缺失 | 缺少必填项 | "缺出院诊断主诊断" |
| 🟠 逻辑冲突 | D1–D8 触发 | "手术日期不在住院期间(D3)" / "患肢侧别在资料间矛盾(D5)" |
| 🟡 术语未规范 | 知识库未收录 | "术式名'打钉'未能规范化，请确认" |
| 🔵 待医生补充 | 原文无此信息、留占位 | "复诊医生及时间原文未见，待补充" |
| 💡 可选建议（非原文） | 知识库通用建议，**未写入正文**，供医生斟酌 | "原文未见分阶段康复医嘱；如需，知识库参考：踝术后0–2周免负重…（此为通用建议，非本例原文）" |

> **忠实性优先**：凡属"原文没有、AI 依知识库/常识补的内容"，一律进 💡（不写入正文），不得当作事实写进出院记录。数值/药名/事件若与原文不符，进 🟠 并标【待核对】。

示例：

> **质控清单**
> - 🔴 必填缺失：住院号、复诊医生及时间
> - 🟠 忠实性/逻辑：无（已逐字核对药名"那屈肝素钙"、Caprini 8 分与病程一致）
> - 🟠 OCR 噪声/来源冲突：传染标志物单位一处为"S/CD"、一处为"S/CO"，已按可核实来源采用"S/CO"，请医生核对；D-二聚体数值疑似箭头误并入，已按原始报告写"3.62↑mg/L"
> - 🟡 术语未规范：内固定物"钢板"未细化型号
> - 🔵 待医生补充：院外随访复查日期原文未见
> - 💡 可选建议（非原文，未写入正文）：踝关节骨折内固定术后分阶段负重/康复方案、VTE 疗程——如需写入请医生确认

## 四、输出约束

- 草稿与质控清单**必出**；JSON 可选（用户要求或入库场景时输出）。
- 不臆造：缺失项一律占位/标 missing，绝不编造。
- 忠实可追溯：JSON 保留 `source` 与 `raw_*`。
- 免责声明必附。
